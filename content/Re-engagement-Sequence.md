# Re-engagement Win-Back Sequence (WeekendMVP pivot)

Built as a draft automation in Beehiiv. Nothing sends until you publish it from the editor.

**Automation:** Re-engagement Win-Back (2 emails)
**Editor:** https://app.beehiiv.com/automations/e1576aa9-33ad-46ee-b9f6-184aef9614d5/workflow
**Trigger:** Subscriber enters the "⚠️ Re-engage — no opens in 90 days" segment (121 people now)

## How the flow works

1. Subscriber enters the Re-engage segment.
2. **Email 1** sends right away.
3. **Wait 3 days.**
4. **Branch:** did they open or click in those 3 days?
   - No (still silent) → **Email 2** sends, then exit.
   - Yes (came back) → exit. They get left alone.

Only the truly dormant get the final notice.

## To go live

1. Open the editor link above.
2. Review both emails and the 3-day wait.
3. Publish. Beehiiv will ask whether to enroll subscribers already in the segment. Choose yes to send to the current 121.
4. After the sequence runs (about a week), open the Re-engage segment. Whoever remains did not respond. Select all and Archive.

## Note on the Shipable workshop date

The emails link to the Shipable page, which always shows the next workshop date, so the copy stays correct over time. If you want to name the specific date (Sat Jun 27, 2026) for this immediate send, add it to the PS before publishing. Remove the hard date again if you keep this automation running long-term.
