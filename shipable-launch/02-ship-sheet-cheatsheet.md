# PART B — FACILITATOR CHEATSHEET *(private — John only)*

## The deal you must deliver
**Every live attendee leaves with a deployed URL.** If that happens, the workshop worked — refunds near zero, real testimonials harvested, DARE bridge earned. Protect this above all else.

## Timing spine (timer on your second screen)
| Clock | Move | Hard stop |
|---|---|---|
| 0:00 | Open + rails | 0:10 |
| 0:10 | M1 Sharpest idea | 0:33 |
| 0:33 | M2 Weekend cut | 1:03 |
| 1:03 | M3 Build + deploy | 1:38 |
| 1:38 | First users + bonuses + DARE tease | 1:45 |
| 1:45 | Q&A + URL wall | 2:00 |

When the timer dings: **move on.** Stragglers get the replay. Momentum > completeness.

## The chat checkpoints (your engagement engine)
1. Top: "your idea in 5 words"
2. End M1: "paste your one-liner" → react to 5–6 *by name*
3. End M2: "your one screen"
4. M3: **you paste your URL first**, then "paste yours"
5. Close: "drop your URL" → **screenshot the chat**

## Live-build safety (most important)
- **Pre-built fallback URL ready in a tab.** If your live build stalls >90s, say "while that generates, here's one I built earlier" and paste the fallback. Never debug on camera for minutes.
- Rehearse the build **once, timed.** If deploy > 5 min, simplify the example.
- Have the **build tool already logged in**, prompt pattern in clipboard, a blank project open.
- Two tabs: build tool + deployed example. One window for slides.

## If things go sideways
- **Build tool down / rate-limited:** switch to the fallback flow, walk them through the prompt anyway, tell them to run it post-call with the replay. Still a win.
- **Quiet room:** call on the chat one-liners by name. Ask a yes/no everyone can answer.
- **Someone's way ahead:** make them a hero — "drop your URL, show the room."
- **Someone's stuck on idea:** give them your example to clone and rename. Shipping beats originality today.
- **Running long:** cut the feature-funeral discussion and Q&A depth, never the build.

## Don'ts
- Don't hard-sell DARE. One sentence, slide 26, then the email does the work.
- Don't read slides. They're rails.
- Don't let M1/M2 eat the build. The URL is the product.
- Don't promise outcomes ("you'll get users/make money"). Promise the *deliverable* (a live URL + a plan).

## Post-call, within 3 hours
- [ ] Send replay + DARE bridge email (E8) to all buyers
- [ ] Save chat screenshot + any URLs people shared → proof wall
- [ ] Note who shipped (for testimonial asks in E9)
- [ ] Refund anyone who asks, instantly
