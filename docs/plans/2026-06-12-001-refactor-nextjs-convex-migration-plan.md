# refactor: Migrate Weekend MVP to Next.js 16 + Tailwind v4 + Convex + shadcn/ui

## Summary

Re-platform the Weekend MVP site from static HTML/Tailwind v3/build-time markdown into a Next.js 16 App Router app with Tailwind v4, shadcn/ui, and Convex as the metadata/event database — without losing any of the 141 indexed URLs, JSON-LD surfaces, OG image paths, or behavioral conventions (email gate, Beehiiv enrollment, Stripe webhook). The execution model is **slice-by-slice via Vercel `fallback` rewrites**: a fresh Next.js project at `next.weekendmvp.app` proxies everything it does not own to a legacy subdomain serving the current static deploy, and each shipped slice (marketing → articles → newsletter → ideas → programmatic SEO → shipable) silently takes over its routes. Cutover swaps DNS once every slice is verified.

Editorial bodies live as MDX-on-disk under `content/`. Convex owns structured metadata, subscription/Stripe events, and is pre-shaped to accept a future `users` + `saved_ideas` layer (Clerk + `ConvexProviderWithClerk`) without re-architecting the schema. Build-time OG generation (Recraft → Satori → sharp) is preserved and outputs to `public/og/...` to keep social-card URLs stable. The 24 nav/sync/publish Node scripts retire — Next.js layouts and Convex queries do their work.

---

## Problem Frame

The current site is a hand-stitched static build: 10 top-level HTML pages, 78 idea pages, 19 articles, 11 newsletter issues, 19 programmatic-SEO landing pages, 5 Vercel Edge functions, ~5,000 lines of `partials/` + `scripts/` glue that keeps shared nav, OG cards, JSON-LD, analytics consent, and inline GA/Pixel snippets in sync across every file. Authoring a new idea requires running 4-6 sync scripts. Inline gtag + Meta Pixel snippets are duplicated across 90+ files. Tailwind Play CDN still loads alongside compiled CSS on idea pages. There is no data layer — `ideas/manifest.json` (3,432 lines) is the source of truth, and the `startup-ideas` filter UI is hand-baked HTML regenerated by `update-startup-ideas.js`.

This shape blocks every future direction the product wants: user accounts and dashboards, saved-ideas, comments, search, personalization, recommendation, A/B testing, real-time anything. It also makes routine edits high-friction and accumulates duplication. Migrating to Next.js + Convex resolves both the present authoring friction and the future product-platform requirement in a single move, while preserving the SEO equity that took two years to build.

## Requirements

- R1. Every URL in `sitemap.xml` (141 entries across `/ideas/`, `/articles/`, `/newsletter/`, `/ideas-for/`, `/build-with/`, `/solve/`, root pages) continues to resolve with status 200 throughout and after migration; no path renames without a 301.
- R2. Existing OG image URLs (`/image/og/idea/{slug}.png`, `/image/og/article/{slug}.png`, `/image/og/newsletter/{slug}.png`) remain at the same paths to preserve Twitter/LinkedIn/Slack cached cards. The default `/image/og-image.png` referenced by all current idea pages remains a working path.
- R3. JSON-LD structured data (`CollectionPage`, `ItemList`, `FAQPage`, `BreadcrumbList`, `Article`, `SoftwareApplication`, `HowTo`, `Person`) is server-rendered on each migrated page from the same data source that drives the page content.
- R4. Email gate behavior preserved: idea page body content is **server-rendered visible by default**; the gate overlay is applied **only on the client** after hydration based on a localStorage probe. AI crawlers (GPTBot, ClaudeBot, PerplexityBot) and Google must continue to see full content.
- R5. Beehiiv subscription routes (`/api/subscribe`, `/api/ideas-subscribe`, `/api/ideas-verify`) port verbatim including the `ALLOWED_AUTOMATIONS` and `ALLOWED_UTM_CAMPAIGNS` allowlists; the paid-cohort automation `aut_b55c6b1e-...` is enrollable **only** from the Stripe webhook server-side.
- R6. Stripe webhook (`/api/stripe-webhook`) continues to verify HMAC-SHA256 signatures with 300s tolerance, returns 200 even on Beehiiv enrollment failure (to suppress Stripe retry loops), and writes a `stripe_events` row to Convex for downstream durability.
- R7. `/ideas/today` continues to redirect to the most recently published idea, sourced from Convex `ideas` table (not `manifest.json`).
- R8. Accessibility conventions in `CLAUDE.md` (icon-only buttons require `aria-label`, decorative icons require `aria-hidden`, focus rings `focus:ring-white/40` / `focus:ring-black/30` minimum, logo divs `role="img" aria-label="Weekend MVP"`) are encoded in typed shadcn wrapper components so violations fail TypeScript compilation.
- R9. Cookie consent flow (`analytics_consent` localStorage key, 365-day expiry, accept/reject/customize banner, conditional gtag `G-Z1NYERTKRS` + Meta Pixel `1602726847528813` loading) is unified into one React context provider; per-page inline duplication of the consent + analytics snippet is eliminated.
- R10. Convex schema shape now accepts a future `users` table (`tokenIdentifier`, `email`, optional `stripeCustomerId`) and `saved_ideas` join table (`userId`, `ideaId`, `savedAt`) without modifying the `ideas` schema. No auth is wired in this migration.
- R11. Authoring a new idea no longer requires running multiple sync scripts. New ideas are added by (a) writing one MDX file under `content/ideas/`, (b) running a single Convex mutation to insert metadata, OR by a single `pnpm publish:idea` script that does both. The current `partials/` + `scripts/sync-*.js` pipeline is retired.
- R12. Newsletter authoring continues to consume markdown from `content/newsletter/2026/{MM}/week-{NN}/*.md` with existing frontmatter shape (the `.claude/newsletter` skill's contract is preserved).
- R13. Tailwind Play CDN script tags are removed from every migrated page; all styling resolves from the Next.js build.
- R14. Hard-coded Beehiiv publication ID, form ID, and automation IDs move to environment variables (`BEEHIIV_PUBLICATION_ID`, `BEEHIIV_DEFAULT_FORM_ID`, `BEEHIIV_DEFAULT_AUTOMATION_ID`, `BEEHIIV_SHIPABLE_AUTOMATION_ID`, `BEEHIIV_IDEAS_AUTOMATION_ID`).
- R15. The `subscribe/route.js` dead-code shadow file is not carried forward.

## Scope Boundaries

- No design or copy changes. Pixel parity with the current site is the bar at the moment each slice cuts over.
- No user accounts, dashboards, auth, sign-in, profile, or any new product surface in this migration. The schema is pre-shaped to accept them later.
- No replacement of Beehiiv as the email engine. The Beehiiv Send API stays unused (Enterprise-only; was attempted and removed 2026-04-19 per `BEEHIIV_CURSOR_RULES.md`).
- No host change. Vercel remains the platform.
- No localization, RSS feed, on-site search, dark/light theme toggle, or any feature not in the current site.
- No migration of `.claude/` skills, MCP servers, or publish scripts under `.gstack/`. Those remain authoring-side tools.
- No replacement of build-time OG generation with runtime `ImageResponse`. The current Recraft → Satori → sharp pipeline (`lib/og/`) is preserved; outputs move to `public/og/`.
- No migration of `test-api.html` (internal smoke-test page) into production. It is excluded from the new build.

### Deferred to Follow-Up Work

- **Convex Auth or Clerk integration** for users + saved ideas: separate PR after migration is complete and DNS has cut over. The schema reservation in U2 is sufficient pre-staging.
- **HTML→MDX one-time extraction quality pass** for the 88 idea pages without structured source: U9 runs the extraction; a follow-up editorial pass to clean up the auto-extracted MDX is out of scope here.
- **Seeding `docs/solutions/`** with bug/pattern entries learned during this migration: opportunistic, not gating.
- **Retiring the legacy subdomain** at the end: separate trivial PR after a stabilization window post-cutover (see U14).

## Key Technical Decisions

- **Two-Vercel-project slice pattern.** A fresh Vercel project deploys the Next.js app at `next.weekendmvp.app` (DNS pre-staged). The existing static site remains at `weekendmvp.app` throughout the slicing phases. The Next.js app's `next.config.ts` `rewrites().fallback` proxies all unmatched paths to `https://legacy.weekendmvp.app` — a temporary alias of the existing static deploy. Cutover (U14) is a DNS swap: `weekendmvp.app` → Next.js project, `legacy.weekendmvp.app` retired after a 30-day stabilization window. Rationale: keeps blast radius zero during slicing, makes rollback trivial (DNS revert), and avoids hand-maintaining 100+ explicit rewrite rules.
- **MDX-on-disk for bodies, Convex for metadata.** Editorial bodies live under `content/ideas/{slug}.mdx`, `content/articles/{slug}.mdx`, `content/newsletter/2026/{MM}/week-{NN}/{slug}.md` (existing structure). Convex `ideas` / `articles` / `newsletter_issues` tables hold metadata (slug, title, publishedAt, category, tools, audiences, scores, og status). Pages join MDX body + Convex metadata at build/render time. Filter UIs (`/startup-ideas`, `/ideas-for/{audience}`, `/build-with/{tool}`) become Convex queries with cached prerendering. Rationale: Git-tracked editorial content with PR review and rollback; Convex realtime ready for future user state without re-architecting.
- **Pre-stage future auth schema, do not wire auth.** `users` and `saved_ideas` tables are declared in `convex/schema.ts` with optional fields and indexes; nothing reads or writes them in this migration. Future auth is Clerk + `ConvexProviderWithClerk` (Convex Auth is still beta and has Next.js middleware friction). Rationale: makes the future-auth migration a wiring problem, not a schema problem.
- **Stripe webhook moves to Node runtime + `stripe` SDK.** Current code uses Edge + hand-rolled HMAC. Node runtime + `stripe.webhooks.constructEvent(rawBody, sig, secret)` reduces custom crypto surface area, gives typed event objects, and Fluid Compute removes the cold-start argument for staying on Edge. The 200-on-Beehiiv-failure behavior and 300s tolerance preserve verbatim. Rationale: SDK ergonomics + standard pattern, no production downside under Fluid Compute.
- **Build-time OG generation preserved.** The `lib/og/` pipeline (Recraft v3 background → Satori chrome → sharp PNG) is not replaced with `next/og` `ImageResponse`. OG outputs move from `image/og/...` to `public/og/...` so existing indexed URLs resolve unchanged. New pages reference OG paths via `generateMetadata`'s `openGraph.images`. Rationale: URL stability for cached social cards; the existing pipeline already supports 5 surface types with Recraft → OpenAI fallback.
- **Cache Components on day one.** `cacheComponents: true` in `next.config.ts` from the bootstrap PR. Pages opt into caching with `use cache` + `cacheLife('hours')` + `cacheTag('idea:' + slug)`. Convex changes trigger Next.js cache invalidation via a Convex HTTP action that POSTs to `/api/revalidate?tag=idea:<slug>&secret=<env>` which calls `revalidateTag(tag, 'max')`. Rationale: avoids a second migration from PPR experimental to stable; matches Next.js 16's current direction.
- **Typed accessibility wrappers around shadcn primitives.** A `<IconButton>` wrapper requires `aria-label` as a TS-required prop. A `<Logo>` wrapper sets `role="img" aria-label="Weekend MVP"` by default. A `<NavExternalLink>` wrapper appends the `(opens in new tab)` sr-only span. Rationale: makes the CLAUDE.md a11y rules unviolatable at compile time.
- **One canonical mega-nav + one idea-page nav.** The current 3 nav variants (mega, idea, reading, starter-kit) collapse to 2 (mega-nav for marketing/listing pages, idea-nav for idea-detail-style reads) in U3 to address QA-flagged inconsistency. `nav-starter-kit.html` becomes a sub-nav slot inside the standard mega-nav layout. Rationale: rewrite boundary should not create visible layout shift; reconciling now is cheaper than later.
- **No code-pull from `subscribe/route.js`.** Dead-shadowed `api/subscribe/route.js` (never wired into any page) is not ported; only `api/subscribe.js` semantics carry forward.

## Open Questions

### Resolved During Planning

- **Edge vs Node for Beehiiv subscribe routes?** Stay on Edge runtime for all 3 Beehiiv routes (`subscribe`, `ideas-subscribe`, `ideas-verify`). Fluid Compute supports both; Edge keeps round-trip latency low for the global CORS-allowlisted form posts. Stripe webhook moves to Node (decision above).
- **`next-mdx-remote` or `@next/mdx`?** Use `next-mdx-remote-client/rsc` (the maintained fork; original is archived). Frontmatter via `parseFrontmatter: true`. Syntax highlighting via `rehype-pretty-code` (Shiki). Plain markdown for newsletter (existing pipeline already produces Tailwind-classed HTML; we re-render via MDX components in U8).
- **How to seed Convex from `manifest.json`?** One-time seed script (`pnpm seed:convex`) reads `ideas/manifest.json`, `articles/manifest.json`, `newsletter/manifest.json` and inserts via `internal.seed.bulkInsert*` Convex mutations. Re-runnable (upserts by slug). Lives in U9.
- **Trailing slashes and `.html`?** Pick extension-less, no-trailing-slash as canonical (matches `_template.html`'s canonical URL pattern). U13 emits 301 redirects for both `.html` and trailing-slash variants from `next.config.ts` `redirects()`.

### Deferred to Implementation

- **Convex index design for `startup-ideas` faceted filtering.** Working hypothesis: primary `by_publishedAt` + `by_category_score` indexes for the common cuts; arrays (`tools`, `audiences`) use either a search index with `filterField`s or a denormalized join table. Implementation will pick after seeing real query patterns in U10/U11.
- **Whether to extract idea-page bodies via cheerio or accept Convex-stored block JSON for the 88 unstructured ideas.** U9 will try cheerio extraction against the 7-section contract (`audit-ideas.js` knows it). Quarantine the ones that fail audit; fall back to Convex `body` field for those.
- **Exact Tailwind v4 utility renames required in compiled CSS.** Run `npx @tailwindcss/upgrade` in U1 and triage what it flags; no preemptive guessing.
- **Whether `generateImageMetadata` is needed anywhere** (for multi-locale or multi-variant OG). Not in scope today; U13 uses static `public/og/...` files.

## System-Wide Impact

- **Interaction graph:** All 5 existing API endpoints touched (subscribe, ideas-subscribe, ideas-verify, ideas-today, stripe-webhook). Beehiiv API integration touched (still external; calling code re-shaped). Stripe webhook handling touched (Edge → Node runtime). Convex is new — every page render and every form submit can write to it.

Stripe webhook failures must NEVER 5xx to Stripe — always 200 (current behavior preserved). Convex write failures during subscribe should not block the user response — they're for our analytics, not for the user's outcome.

  - Email gate localStorage key `ideas_email` — the existing key continues to be honored so previously-subscribed users are not re-gated.

- **API surface parity:** All 5 Vercel Edge endpoints port to Next.js Route Handlers with identical request/response contracts. Existing form submitters (including any third-party Beehiiv embed scripts) continue to work unchanged.
- **Integration coverage:** End-to-end form-submit → Beehiiv enroll → Convex log → next-page-render-honors-state needs explicit integration tests (cannot be proven by unit mocks). End-to-end Stripe webhook → Beehiiv paid enrollment must be tested with `stripe trigger`.

  - URL space (141 paths) is unchanged in canonical form.

## Risks & Dependencies

| Risk | Mitigation |
|------|------------|
| 88 idea bodies lack structured source; HTML→MDX extraction may lose formatting fidelity (callouts, sidebars, inline highlights). | Extraction script (U9) uses the section contract from `audit-ideas.js`. Quarantine ideas that fail audit; render those from a Convex `body` field instead of MDX. Spot-check 5 random extractions before committing. Accept that some legacy ideas may need manual MDX touch-up post-migration; track via a follow-up issue, not a migration blocker. |
| OG image URL changes break cached social cards. | Move `image/og/` → `public/og/` and emit 301 redirects from the old paths. Twitter/LinkedIn re-fetch on next reference. The default `/image/og-image.png` path is preserved at `public/image/og-image.png`. |
| Email gate regression: if EmailGate accidentally renders server-side, crawlers see no content and SEO/AEO collapses. | Failing test for R4 written before the EmailGate component (Execution note on U10). Test verifies the raw HTML response (no JS executed) contains full idea body. CI gates on this test. |
| Stripe webhook signature path: moving from hand-rolled Web Crypto to Node `stripe.webhooks.constructEvent` could miss an edge case (e.g., legacy signatures, replay protection). | Replay U5 tests against staging Stripe test events. Keep the 300s tolerance. Idempotency by `stripeEventId` covers retry storms. |
| Convex schema decision (single faceted search index vs. denormalized join tables for `tools`/`audiences`) is deferred to implementation; wrong choice surfaces as slow `/startup-ideas` filter UI. | Index decision deferred per Open Questions. Re-evaluate after U11 ships; switch to denormalized join tables if the search index pattern is too slow on a 1k-row table. |
| Tailwind v4 utility renames could cause visual drift across 100+ pages. | Run `npx @tailwindcss/upgrade` in U1 and commit the resulting changes atomically. Visual regression testing on the marketing slice (U6) before proceeding. |
| Vercel `fallback` rewrite + Next.js redirect interaction: a `.html` URL hitting redirects first then NOT matching might still proxy to legacy `.html`. | Redirects apply before rewrites in Next.js; test explicitly. The redirect chain `/foo.html → /foo` works even if `/foo` is a Next.js route. Validate with `curl -I` against staging. |
| Convex Auth beta blocks the future-user-accounts addition. | Schema is pre-staged with `users` + `saved_ideas` declared. Future migration is Clerk + `ConvexProviderWithClerk` (Convex's documented recommendation) — purely additive, no schema change. |
| `legacy.weekendmvp.app` aliasing requires a temporary domain on Vercel that resolves to the current production deploy. | Add `legacy.weekendmvp.app` as a domain alias on the current Vercel project before the U1 deploy. Both `weekendmvp.app` and `legacy.weekendmvp.app` serve the same content during slicing. |
| Newsletter authoring contract changes accidentally. | R12 is explicit. U8 reads the existing `.md` files at runtime without transformation; the `.claude/newsletter` skill's output is the same on both sides of migration. |
| Hidden Tailwind utilities on idea pages (from the Play CDN's JIT) may render differently when only the build output is loaded. | U10 visual diff sample of 5 idea pages catches this. If gaps surface, add the missing utility classes to the build content scan. |
| `manifest.json` reference tables (`audiences[]`, `categories[]`, etc.) contain rich landing-page copy embedded inside the JSON. Lossy migration could lose programmatic SEO copy. | Seed script preserves every field as-is into Convex reference tables. Landing pages in U11 read those fields. |
